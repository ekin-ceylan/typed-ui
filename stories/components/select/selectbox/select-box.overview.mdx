# Select

> Seçenek sayısı çok fazla değilse tercih edilebilir. Seçenek sayısı 10'dan fazlaysa veya seçenek listesi sayfa boyunu aşıyorsa `ComboBox` tercih edilmelidir.

`select-box`, formlarda veya kullanıcı arayüzlerinde seçenekler arasından seçim yapılmasını sağlayan, erişilebilir ve özelleştirilebilir bir seçim kutusu bileşenidir. Hem klasik `<select>`/`<option>` slot kullanımıyla hem de programatik olarak seçenek eklemeye olanak tanır.

Doğrulama, hata mesajı, klavye ve ekran okuyucu desteği gibi modern form ihtiyaçlarını karşılamak üzere tasarlanmıştır. Tasarım token’ları sayesinde kolayca özelleştirilebilir ve farklı temalarla uyumlu çalışır.

## Kurulum ve Temel Kullanım

### Kurulum

`select-box` bir **Web Component (Custom Element)** olduğu için kullanılmadan önce _register_ edilmelidir. Bunun için, `typed-ui` paketini projenize ekledikten sonra uygulamanızın veya sayfanızın giriş noktasında (ör. `main.js`, `page.js`), `-` içeren uygun bir bileşen ismi kullanarak, aşağıdaki kodlardan birini çalıştırın:

```js
import { SelectBox } from 'typed-ui';

customElements.define('select-box', SelectBox);
```

veya

```js
import * as TypedUI, { defineComponent } from 'typed-ui';

defineComponent('do-select', TypedUI.SelectBox);
```

ESM build yerine IIFE build kullanıyorsanız.

```js
TypedUI.defineComponent('do-select', TypedUI.SelectBox);
```

## Erişilebilirlik

### Etiketleme ve İlişkilendirme

Görünür etiket kullanımında `<label for="..."></label>` ile `<select id="..."></select>` ilişkilendirmesi yapılır. Bu durumda `aria-labelledby` otomatik bağlanır.

`hide-label` kullanıldığında görünür `<label>` yerine `aria-label` kullanılır.

### Required ve Hata Durumu

`required` verildiğinde `aria-required="true"`, aksi durumda `aria-required="false"` atanır.

Alan geçersiz olduğunda `aria-invalid` atanır ve hata metni gösterilir. `aria-errormessage`, hata alanının id değerine bağlanır. Değer tekrar geçerli olduğunda `aria-errormessage` kaldırılır.

### Klavye Davranışı

Native `<select>` davranışı korunur (özellikle seçenek gezinimi tarayıcı tarafından yönetilir). Bileşen seviyesi açık/kapalı durumu için `Enter`/`Space` ile açma, `Escape`/`Tab`/`Enter` ile kapama davranışı uygulanır.

## Programatik Kullanım

> `options` set edildiğinde, slot ile verilen `<option>`/`<optgroup>` içerikleri yok sayılır.

Seçenekleri JS üzerinden de verebilirsiniz. Bu durumda **slot içeriği yok sayılır**.

`options`, ya bir `string` dizisi olmalıdır ya da elemanları aşağıdaki arayüzlerden birini sağlamalıdır:

```ts
type Option = {
    value?: string;
    label?: string;
    text?: string;
    selected?: boolean;
    disabled?: boolean;
    hidden?: boolean;
    dataset?: Record<string, string | number | boolean | null | undefined>;
    ariaset?: Record<string, string | null | undefined>;
};

type OptionGroup = {
    label?: string;
    disabled?: boolean;
    hidden?: boolean;
    dataset?: Record<string, string | number | boolean | null | undefined>;
    ariaset?: Record<string, string | null | undefined>;
    options?: Option[];
};

options: Array<string | Option | OptionGroup>;
```

Öncelikle sayfada var olan `SelectBox` elementini seçmeniz

```js
const el = document.getElementById('#basic-slot');
```

veya yeni bir `SelectBox` oluşturmanız gerekir.

```js
const el = new SelectBox();
body.appendChild(el);
```

daha sonra seçeneklerinizi, aşağıdaki kod örneklerinde görüldüğü gibi ekleyebilirsiniz.

```js
el.options = [
    { value: 'tr', label: 'Türkiye' },
    { value: 'de', label: 'Almanya', selected: true },
    { value: 'us', label: 'ABD', disabled: true },
];
```

`dataset` ve `ariaset` ile option seviyesinde özel öznitelik geçebilirsiniz:

```js
el.options = [
    {
        value: 'ank',
        label: 'Ankara',
        dataset: { trackingId: 'city-ank' },
        ariaset: { label: 'Ankara seçeneği' },
    },
];
```

`<optgroup>` desteği:

```js
el.options = [
    {
        label: 'Avrupa',
        dataset: { region: 'eu' },
        ariaset: { label: 'Avrupa ülkeleri' },
        options: [
            { value: 'de', label: 'Almanya' },
            { value: 'fr', label: 'Fransa' },
        ],
    },
    {
        label: 'Asya',
        options: [
            { value: 'tr', label: 'Türkiye', selected: true },
            { value: 'jp', label: 'Japonya' },
        ],
    },
];
```

> `dataset` ve `ariaset` anahtarları _prefix_ kullanılmadan verilmelidir:
>
> - `dataset.testId` -> `data-test-id`
> - `ariaset.label` -> `aria-label`.

### Kapsam Dışı Davranışlar

- Tüm öznitelikler için sınırsız _passthrough_ desteklenmez.
- `onclick` gibi `on*` event öznitelikleri model üzerinden geçirilmez.
- `style` model üzerinden geçirilmez.

### Olay Dinleyiciler (_Events_)

`select-box` aşağıdaki event'leri bubble edecek şekilde dispatch eder:

- `input`: seçim değiştiğinde
- `change`: özellikle temizleme (clear) sonrası da tetiklenir
- `clear`: temizleme butonuna basıldığında
- `update`: `value` programatik değiştiğinde
- `validate`: doğrulama çalıştığında (`detail.validationMessage` içerir)

```js
const el = document.querySelector('select-box[field-id="country"]');

el.addEventListener('input', e => {
    console.log('value:', el.value, 'detail:', e.detail);
});

el.addEventListener('clear', () => {
    console.log('cleared');
});
```
